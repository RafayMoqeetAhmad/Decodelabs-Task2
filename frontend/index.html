API Documentation

All endpoints exposed by the backend — http://localhost:3000

GET
/api/status

Health check. Returns server status, timestamp, and all available endpoints.

{
  "success": true,
  "data": {
    "status": "online",
    "totalUsers": 4
  }
}
GET
/api/users

Returns all users. Optional query params: ?role=intern, ?search=rafay

{
  "success": true,
  "data": {
    "count": 4,
    "users": [...]
  }
}
GET
/api/users/:id

Returns a single user by ID. Returns 404 if not found.

{
  "success": true,
  "data": {
    "user": { "id": 1, ... }
  }
}
POST
/api/users

Creates a new user. Validates name, email format, and role. Returns 201 on success.

Body: {
  "name": "Ali Khan",
  "email": "ali@test.com",
  "role": "intern"
}
PUT
/api/users/:id

Updates an existing user. Partial updates supported. Returns 404 if not found.

Body: {
  "name": "Updated Name",
  "role": "mentor"
}
DELETE
/api/users/:id

Permanently deletes a user by ID. Returns the deleted user object.

{
  "success": true,
  "data": {
    "deleted": { "id": 2, ... }
  }
}

HTTP Status Codes

200OK
201Created
400Bad Request
404Not Found
409Conflict
500Server Error
